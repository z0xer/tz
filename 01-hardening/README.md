# Базовая защита сервера (Ubuntu/Debian)

Скрипт для первичной настройки чистого сервера. Закрывает основные дыры безопасности, чтобы сервер не взломали автоматическими сканерами в первый же день.

## Что делает скрипт

* Обновляет систему и сохраняет списки пакетов до/после.
* Настраивает SSH: меняет стандартный порт на 2222, запрещает авторизацию по паролю и полностью отключает доступ для root.
* Включает файрвол (UFW): запрещает все входящие соединения, кроме портов 2222 (для SSH) и 8080 (для веба).
* Создает рабочего пользователя dev. Выдает ему права на нужные команды (systemctl, docker, apt) и настраивает папку для ключей.
* Включает автоматическую установку критических обновлений безопасности.
* Жестко отключает лишние службы (печать, bluetooth, avahi), чтобы не тратили ресурсы и не создавали уязвимостей.
* Ставит fail2ban для защиты от перебора паролей (банит айпишник на 30 минут после 3 неудачных попыток).
* Включает auditd, чтобы писать в лог все изменения конфигов SSH, файла паролей и использования команды sudo.

## Важно перед запуском

Обязательно откройте скрипт в редакторе и впишите свой публичный ssh-ключ. Если этого не сделать, после работы скрипта вы навсегда потеряете доступ к серверу.

Найдите в скрипте эту строку:
`echo "мой pub_key" > /home/dev/.ssh/authorized_keys`

И замените `"мой pub_key"` на ваш реальный открытый ключ (обычно начинается с ssh-rsa или ssh-ed25519).

## Запуск

Закидываем скрипт на сервер, выдаем права на выполнение и запускаем от имени root:

```bash
chmod +x harden.sh
sudo ./harden.sh
```

## Как подключаться после настройки

После выполнения скрипта зайти по паролю или под рутом будет нельзя. Порт тоже изменится. Подключаться нужно так:

```bash
ssh -i ~/.ssh/ваш_приватный_ключ -p 2222 dev@ip_адрес_сервера
```
