# Отчёт по утечкам секретов

Инструмент: gitleaks
Репозиторий: https://github.com/trufflesecurity/test_keys

## Найденные утечки

| Тип секрета      | Файл        | Критичность |
|-----------------|-------------|-------------|
| AWS Access Key  | keys/aws    | Критическая |
| AWS Secret Key  | keys/aws    | Критическая |
| RSA Private Key | keys/rsa    | Критическая |




## Что делать если секрет утёк (5 шагов по порядку)
Убить -> Проверить -> Заменить -> Вычистить -> Защитить

1. отозвать секрет — зайти в AWS Console / GitHub Settings и деактивировать
   утёкший ключ. Создать новый.

2. чекнуть логи — посмотреть в CloudTrail (AWS) или Audit Log (GitHub)
   не использовал ли кто-то этот ключ. Если есть какиеөто следы то это уже инцидент 

3. обновить ключи  — новый ключ прописать во всех сервисах, CI/CD, Vault.
   Старый уже не работает после шага 1.

4. почистить гит — удалить секрет из истории коммитов через BFG Repo-Cleaner
   или git filter-repo. Потом git push --force.

5. предотварить — поставить pre-commit хук с gitleaks,
   добавить проверку в CI/CD, провести ревью с командой.
   Еще лучше добавить sast check
